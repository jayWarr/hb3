execute the utiliity

This hummingBird utility harvests poem titles from their document files.


Important In order for the utility to work your poems must be in individual files and each file must have the title as the first line.  The utility can read the following document file types:


    •  .txt
    • .doc
    • .docx
    • .odt
    • .rtf
    • .rtfd
    • .pdf
    • .ipage


The utility is executed in a terminal window with a number of optional controls;

It was built for speed (it is very fast typically scanning 1000's of files in a few seconds) and ubiquity.

The same code  executes across all platforms. The utility creates a tamper proof text upload file of titles.


Important don't attempt to alter the upload file -  if you do it won't work!


How to use

    1. Download the correct executable for your operating system. There are different versions of the ulitity for Windows, OSX & Linux.
    2. Move the file to the top level directory that contains your poem documents

(strictly this is not necessary - you can locate the file where ever you wish - it just makes               specifying the directory and sub-dirs to scan easier)

    1. Open a terminal window
    2. Navigate to the directory in 2
    3. type in the command __hb_gpt_upload
    4. Your are going to be met with a series of prompts for input - it you respond with [RET] to each one the utility will take a sensible default
    5. If you execute the utility like this: __hb_gpt_upload -h        you'll get a verbose list of the options all the options can be abbreviated to a single letter or extended to a descriptive word.

Examples


jr@Orac poetry % ___hb_gpt_upload -h

 

usage: hb_gpt_upload_2 [-h] [-u] [-v] [-s] [-t] [-d D] [-i] [-e E] [-p P]

                       [-r R] [-b B] [-l L] [-f]


This hummingBird utility extracts poem titles from the filenames that reside

in the document (sub-)directory and builds an upload file of the titles for

the main web application.


optional arguments:

  -h, --help                          show this help message and exit

  -u, --usage                 provide details and instruction on useage, then exit

  -v, --version                provide details and exit

  -s, --scan                          provide initial counts and exit

  -t, --terse                          suppress all unnecessary info. - the default is verbose

  -d D, --dir D                       the path to the document (sub-)directory - the current

                                       working (sub-)dir

  -i, --include                        all the sub-dirs of the document (sub-)dir - N to

                                       exclude subdirs

  -e E, --extn E              filter files by a single extn E - ALL extns , choices

                                      are: .txt|.doc|.docx|.odt|.rtf|.rtfd|.pdf|.ipage

  -p P, --previous P           select files by date last modified in the period P, ALL

                                       files, choices are: [1-10]d | [1-4]w | [1-12]m where

                                       d=days, w=weeks & m=months

  -r R, --reject1st R          reject the first R files - 0, choices 1-1000

  -b B, --block B            list blocks of B titles at a time -

                                       choices for B are: 10|20|50|100

  -l L, --limit L                     limit the selection to L titles - 2000 choices for L

                                      are: 100|200|500|1000|2000

  -f, --filter                        will be prompted for a filter which must be a regular

                                       expression - see usage for examples
